# Invoyr UI — Design Sync Notes

## Repo shape

- Package shape (no Storybook). Components live in `src/components/ui/*.tsx`.
- No built `dist/` — converter runs in synth-entry mode via `--entry ./src/components/ui/index.ts`.
- All 15 components authored with rich previews; all graded good.

## Re-sync command

```bash
# 1. Rebuild the Tailwind CSS tokens file
node .design-sync/build-css.mjs

# 2. Re-run the converter (synth-entry, no --remote on first sync; use remote on re-sync)
node .ds-sync/package-build.mjs \
  --config .design-sync/config.json \
  --node-modules ./node_modules \
  --entry ./src/components/ui/index.ts \
  --out ./ds-bundle

# 3. Validate
node .ds-sync/package-validate.mjs ./ds-bundle

# 4. Capture (only changed/new components)
node .ds-sync/package-capture.mjs --out ./ds-bundle

# 5. Grade any fresh sheets in .design-sync/.cache/review/<Name>.grade.json

# 6. Upload (atomic path — projectId is pinned)
```

## Key setup facts

- **Tailwind v4**: `src/app/globals.css` uses `@import "tailwindcss"` (v4 syntax — not a resolvable file).  
  The `build-css.mjs` script pre-processes it via `@tailwindcss/postcss` into `src/design-sync-tokens.css`.  
  Run `node .design-sync/build-css.mjs` before every converter run.  
  `src/design-sync-tokens.css` is gitignored.

- **Synth-entry**: No `dist/`. Converter uses `--entry ./src/components/ui/index.ts` (barrel file at `src/components/ui/index.ts`). All 15 components listed in `componentSrcMap`.

- **Playwright**: Installed locally (`node_modules/.bin/playwright`) and chromium downloaded to default cache. Render check runs automatically.

- **Overlay components** (Dialog, DropdownMenu, Tooltip): `cardMode: "single"` in overrides. Previews render them with `open` prop set — no user interaction needed.

- **Table**: `cardMode: "column"` (full-width card).

## Re-sync risks

- **Tailwind tokens**: `src/design-sync-tokens.css` is regenerated by `build-css.mjs`. If `globals.css` changes (new CSS variables, new utilities), re-run `build-css.mjs` before the converter — stale tokens produce unstyled components.

- **Barrel file**: `src/components/ui/index.ts` was created for this sync. If new UI components are added to `src/components/ui/`, add them to the barrel and to `componentSrcMap` in `config.json`.

- **Preview content**: Previews use realistic invoice content (`#INV-0042`, `Acme Corp`, GBP). If the domain data model changes (e.g. currency or status labels) the previews may lag. Update `.design-sync/previews/<Name>.tsx` and re-run capture.

- **Playwright version**: Pinned to the `playwright` version in `package.json`. If that version updates without a matching chromium install, the render check fails `[RENDER_SKIPPED]`. Run `npx playwright install chromium` after any playwright update.

- **CVA variant labels**: `Button` and `Badge` variants are documented in `conventions.md`. If variants are added or renamed, update `conventions.md` and re-run the build so the README reflects the change.
